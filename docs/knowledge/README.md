# 游戏知识文档分层

## 三层结构

1. 页面索引层
- 作用：回答“资料在哪一页”。
- 内容：栏目、标题、原始链接、更新时间。
- 文件：`indexes/baike-page-index-2026-03-31.md`

2. 逐页摘要层
- 作用：回答“这一页大致讲什么、哪些信息值得项目关注”。
- 内容：逐页摘要、要点提炼。
- 文件：`summaries/baike-page-summaries-2026-03-31.md`

3. 项目规则层
- 作用：回答“项目真正应该读取哪些字段和规则”。
- 内容：结构化字段、规则候选、建模建议。
- 文件：`rules/game-rule-seeds-2026-03-31.md`

## 使用原则

- 先从索引层定位页面。
- 需要理解规则时进入摘要层。
- 需要写代码、配置、状态机时，优先使用规则层，不直接依赖原文长文本。
- 当规则层缺项时，回到摘要层和原页面补充，而不是让代码直接消费百科长文。
