# 百科重复标题去重策略（2026-03-31）

## 目的

`dh2.baike.163.com` 的资料页存在同名标题重复出现的情况。重复页有几种来源：

- 旧版本页面与新版本页面并存
- 同一主题被挂到多个栏目
- 内容几乎相同，但页面 ID 不同
- 一个页面为空壳，另一个页面是完整版本

本策略的目标不是删除原始页面，而是给项目确定：

- 哪一页作为 `canonical` 主页
- 哪些页面作为 `alias` 或 `legacy` 保留
- 后续索引、摘要、规则层优先引用哪一页

## 判定规则

### 1. 优先保留信息更完整的页面

优先级最高的是内容完整度：

- 正文/摘要非空，优先于空壳页
- 规则更具体，优先于仅有标题或极短说明的页
- 可直接抽字段，优先于只能做占位说明的页

### 2. 完整度相近时，优先保留更新时间更晚的页面

如果两页内容相近，优先保留：

- 更新时间更晚
- 字段更贴近当前版本

### 3. 同名跨栏目时，不按栏目硬判，按内容归属判

例如某些主题被同时挂在 `装备系统` 和 `召唤兽系统` 下。  
判定时不以栏目名为准，而看主题真正属于哪个领域模型。

### 4. 不丢弃原始页面，只做角色区分

每个重复标题页面都保留，但区分为：

- `canonical`：主引用页
- `alias`：内容接近的别名页
- `legacy`：旧版页或空壳页

## 项目使用规则

1. 索引层保留所有页面，不做物理删除。
2. 摘要层允许保留重复页，但后续可以逐步给 `alias`/`legacy` 打标。
3. 规则层只从 `canonical` 主页抽字段，避免同名规则冲突。
4. 后续新增重复标题时，优先按本策略自动判：
   - 先看摘要是否为空
   - 再看规则信息密度
   - 再看更新时间
   - 最后看栏目归属是否更贴近领域模型
